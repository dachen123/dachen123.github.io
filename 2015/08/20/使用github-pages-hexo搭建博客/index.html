    环境的搭建

    起初的参考文章是这，但是它有不少的错别字，以及很多步骤没有解释清楚，导致在安装hexo过程中，始终没法用 hexo s启动本地服务，最后找到了这篇文章，了解到下面这几个指令是不可或缺的：
    npm install hexo –save
    npm install hexo-server –save
    npm install hexo-deployer-git –save
    这是Hexo安装插件的方法，我认为是不是hexo安装以及它的本地服务的安装是分开的，导致之前的失败。这算是我遇到的第一个坑。

    第二个坑是在服务启动后，hexo g 始终无法生成主页，打开http://localhost:4000/ 链接总是报错“cannot GET /”查了半天，原来是在运行hexo之前还要运行下面这条指令：
    npm install
    这能将hexo 缺少的依赖装上，解决这个问题后，主页顺利打开，Oh yeah!

    最后还有这篇文章应该看一下，步骤解释的更为详细

    侧边栏分类和标签的显示

    除了这篇文章提到的，在source目录下建立tags
    和 categories
    文件夹以外，新建博客时千万不要忘了它应该是这个样子的：

    title: postName #文章页面上的显示名称，可以任意修改，不会出现在URL中
    date: 2013-12-02 15:30:16 #文章生成时间，一般不改，当然也可以任意修改
    categories: #文章分类目录，可以为空，注意:后面有个空格
    tags: #文章标签，可空，多标签请用格式[tag1,tag2,tag3]，注意:后面有个空格
    -–
    这里开始使用markdown格式输入你的正文。

    少了categories和tags这两项，主页上是不会把他们显示出来的。
